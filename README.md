PROJECT: AIRBNB CLONE - CONSOLE

DESCRIPTION
This is the first step towards building our first full web application: the AirBnB clone.
In this part of the project, we focus on creating the console, which manages the AirBnB objects. In this project is the first abstracted storage engine of the project: File storage. Also, there is a parent class which parents other classes like User, State, City, Place and Amenities and also take care of the initialization, serialization and deserialization of your future instances.

THE COMMAND INTERPRETER
This feature manages AirBnB objects and it works in both interactive and non-interactive modes. It has commands like create, show, destroy, all, update as well as quit and help. Once the console is started, commands and their arguments can be passed and validated, then processed if valid and if not, there are ways of gracefully handling errors accompanied by them.
